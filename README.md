## About

<div align="center">

**Software engineer focused on shipping production systems end to end — not prototypes.**

</div>

I am a Computer Science student at **LUT University (Lappeenranta–Lahti University of Technology)**, Finland, working across full-stack product engineering, data visualization, and AI-assisted software systems. My work is defined by a single principle: every project ends as something a real user can open, use, and depend on.

- **Software Engineering** — TypeScript/Node.js service design, REST API architecture, relational and document data modelling, authentication and authorization, modular library design, and clean separation of concerns across layered codebases.
- **AI / ML** — Applied LLM integration in production products, prompt and context engineering, retrieval-oriented feature design, and academic work on explainability (XAI) and the black-box trust problem in software engineering tooling.
- **Full-Stack Development** — React + Vite and Next.js frontends, Express/TypeScript backends, PostgreSQL and MongoDB persistence layers, Vercel deployment pipelines, and payment/VAT-compliant billing integration.
- **Product Engineering Mindset** — I scope for a real market segment first, then engineer to it: Finnish SMB accounting workflows, community dietary classification, national electoral data transparency.

## Featured Projects

<details>
<summary><b>Vaalit — Finnish Election Data Visualization Platform</b></summary>

<br>

A client-side visualization platform for Finnish electoral data spanning municipal, parliamentary, presidential and European Parliament elections. Built as a bachelor's thesis artifact under a Design Science Research methodology, with four original shared libraries designed for reuse across election types.

| Attribute | Detail |
| :--- | :--- |
| **Stack** | JavaScript, Leaflet, Chart.js, PxWeb API, HTML5, CSS3 |
| **Scale** | Four election categories, national municipal-level granularity, multi-decade historical coverage |
| **Performance** | Fully client-side rendering, zero backend dependency, lazy dataset loading and cached geometry |
| **Security** | No user data collected, read-only public API consumption, static hosting surface |
| **Impact** | Reusable open library layer (PxLib, MapHelper, ChartLib, elections.js) generalising across election domains |
| **Repository** | [Finnish-Election-Visualize](https://github.com/JusefA/Finnish-Election-Visualize) |

The engineering focus was extraction of a generic core: rather than building four separate visualizers, the system abstracts PX-format data access, geospatial map binding, and chart composition into independent libraries, so adding a new election type becomes configuration rather than reimplementation. The thesis formalises this as a design artifact evaluated against transparency and reusability criteria.

</details>

<details>
<summary><b>Is It Halal — Community Dietary Classification Platform</b></summary>

<br>

A free, non-profit web application for classifying halal dietary information, serving a growing base of active users.

| Attribute | Detail |
| :--- | :--- |
| **Stack** | React, TypeScript, Vercel, structured classification dataset |
| **Scale** | Growing active community user base, continuously expanding ingredient corpus |
| **Performance** | Static-first delivery, sub-second lookups, mobile-first responsive layout |
| **Security** | Zero personal data collection, no authentication surface, fully public read model |
| **Impact** | Fills a genuine information gap for a community underserved by commercial food databases |
| **Repository** | [Live Application](https://isithalal.vercel.app/) |

Built and maintained without monetisation. The engineering constraint was accuracy and speed at zero operating cost, which shaped a static-data architecture over a database-backed one.

</details>

<details>
<summary><b>ItawaPulse — AI Sentiment & Quant Analytics Platform</b></summary>

<br>

A stock screening platform that ranks over 5,000 US equities using a quantitative factor model combined with NLP sentiment analysis of news and social media.

| Attribute | Detail |
| :--- | :--- |
| **Stack** | React, Node.js, Anthropic API, market data APIs, Unusual Whales API |
| **Scale** | 5,000+ US equities scored cross-sectionally on every ranking pass |
| **Performance** | Batched factor computation, incremental sentiment refresh, cached ranking snapshots |
| **Security** | Server-side API key isolation, rate-limited data ingestion, no client-side credential exposure |
| **Impact** | Surfaces smart-money positioning signals normally locked behind institutional tooling |
| **Repository** | [View Repository](https://github.com/JusefA) |

The factor model combines momentum, volatility, and sector-relative fundamentals, then layers NLP sentiment derived from news and social media on top. Real-time options-flow, dark-pool, and congressional-trading data is integrated as a distinct smart-money factor, cross-sectionally scoring net directional premium across every candidate rather than evaluating each ticker in isolation.

</details>

<details>
<summary><b>Finance Copilot — AI Finance Dashboard for Finnish SMBs</b></summary>

<br>

A SaaS finance dashboard targeting small Finnish construction and trades companies, combining bookkeeping visibility with AI-assisted natural-language insight over the company's own financial data.

| Attribute | Detail |
| :--- | :--- |
| **Stack** | Next.js, TypeScript, Supabase, PostgreSQL, Vercel, Paddle, OpenAI |
| **Scale** | Multi-tenant architecture with row-level isolation per company account |
| **Performance** | Server-side rendering, edge deployment, incremental data revalidation |
| **Security** | Magic-link authentication, Supabase Row Level Security, EU VAT-compliant billing via Paddle merchant of record |
| **Impact** | Removes spreadsheet dependency for a segment underserved by enterprise accounting suites |
| **Repository** | [Live Application](https://finance-copilot-web3.vercel.app/) |

Key architectural decisions included migrating from an Electron desktop shell to a web-first application to eliminate distribution friction, and selecting Paddle over direct card processing so that EU VAT obligations are handled at the merchant-of-record layer rather than in application code — a deliberate trade of margin for regulatory surface area.

</details>
